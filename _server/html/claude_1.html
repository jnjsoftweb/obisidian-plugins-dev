    macos 에서 '.cursorrules' 파일을 하위 폴더 전체에서 찾을 때 명령은?

    macOS에서 .cursorrules 파일을 찾기 위한 터미널 명령어는 다음과 같습니다: 

    bash
    find . -name ".cursorrules"

    만약 현재 디렉토리가 아닌 특정 디렉토리에서 검색하고 싶다면, . 대신 해당 경로를 지정하면 됩니다: 

    bash
    find /path/to/search -name ".cursorrules"

    이 명령어는:

    • 숨김 파일도 검색합니다
    • 모든 하위 디렉토리를 재귀적으로 검색합니다
    • 정확히 .cursorrules라는 이름의 파일만 찾습니다

    재귀적으로 모든 디렉토리에서 찾는 건가요?

    네, find 명령어는 기본적으로 재귀적(recursive)으로 동작합니다. 즉, 시작 디렉토리부터 그 아래의 모든 하위 디렉토리까지 전부 검색합니다.

    만약 현재 디렉토리에서만 검색하고 싶다면 (하위 디렉토리는 제외), -maxdepth 옵션을 사용할 수 있습니다: 

    bash
    find . -maxdepth 1 -name ".cursorrules"
    • -maxdepth 1: 현재 디렉토리만 검색
    • -maxdepth 2: 현재 디렉토리와 바로 아래 하위 디렉토리까지만 검색
    • 옵션 없음: 모든 하위 디렉토리 검색

    파일이름에 'cursor'를 포함하는 파일 전체를 찾으려면

    파일 이름에 'cursor'가 포함된 모든 파일을 찾으려면 다음 명령어를 사용할 수 있습니다: 

    bash
    find . -name "*cursor*"

    이 명령어에서:

    • *cursor*: 파일 이름의 어느 위치에든 'cursor'가 포함된 파일을 찾습니다
    • 앞의 *: 'cursor' 앞에 어떤 문자가 와도 됨
    • 뒤의 *: 'cursor' 뒤에 어떤 문자가 와도 됨

    대소문자를 구분하지 않고 검색하려면 -iname 옵션을 사용할 수 있습니다: 

    bash
    find . -iname "*cursor*"

    이렇게 하면 'cursor', 'Cursor', 'CURSOR' 등 대소문자 구분 없이 모두 검색됩니다.

    npx create-next-app@latest 명령에서 설정을 묻지 않고 디폴트 설정으로 설치하려면?

    npx create-next-app@latest 명령에서 모든 설정을 기본값으로 자동 설정하려면 --yes 또는 -y 플래그를 사용하면 됩니다: 

    bash
    npx create-next-app@latest my-app --yes

    또는 더 짧게:

    bash
    npx create-next-app@latest my-app -y

    이렇게 하면 다음과 같은 기본 설정으로 설치됩니다:

    • TypeScript: Yes
    • ESLint: Yes
    • Tailwind CSS: Yes
    • src/ directory: No
    • App Router: Yes
    • Import alias: Yes (@/*)

    만약 특정 옵션만 지정하고 싶다면, 해당 옵션들만 플래그로 지정할 수도 있습니다. 예를 들어: 

    bash
    npx create-next-app@latest my-app --ts --tailwind --eslint

    ## Component Guidelines

### ShadCN Components

- 모든 UI 컴포넌트는 ShadCN을 사용해야 합니다.
- 컴포넌트 사용 전 설치 여부를 확인해야 합니다: `components/ui` 디렉토리 체크
- 컴포넌트 설치 명령어를 사용해야 합니다: `npx shadcn@latest add [component-name]`
  - ⚠️ 주의: `npx shadcn-ui@latest add` 명령어는 deprecated. 반드시 `shadcn@latest add` 사용해야 합니다.
- ShadCN 컴포넌트의 스타일 수정은 `tailwind.config.ts`의 `theme` 설정을 통해 해야 합니다.

### Styling

- 모든 스타일링은 Tailwind CSS를 사용해야 합니다.
- 커스텀 CSS는 최소화하고, 필요한 경우 `app/globals.css`에 정의합니다.
- 반응형 디자인은 Tailwind의 기본 브레이크포인트를 사용합니다: `sm`, `md`, `lg`, `xl`, `2xl`
- 커스텀 클래스는 `className={cn()}` 유틸리티를 사용하여 조합합니다.

### Icons

- 모든 아이콘은 Lucide React를 사용해야 합니다.
- 아이콘 임포트 방법: `import { IconName } from "lucide-react"`
- 예시: `import { Search, Menu } from "lucide-react"`
- 아이콘 크기와 색상은 Tailwind 클래스를 통해 제어합니다.

### Component Structure

- 컴포넌트는 `/components` 디렉토리 내에 위치해야 합니다.
- UI 컴포넌트는 `/components/ui` 디렉토리에 위치해야 합니다.
- 페이지별 컴포넌트는 `/app` 디렉토리 내 해당 라우트에 위치해야 합니다.
- 재사용 가능한 레이아웃 컴포넌트는 `/components/layouts`에 위치해야 합니다.
- 공통 훅은 `/hooks` 디렉토리에 위치해야 합니다.

## Best Practices

- TypeScript 타입은 반드시 정의해야 합니다.
  - 인터페이스/타입은 컴포넌트 파일 내에 정의하거나, 복잡한 경우 별도의 `types` 디렉토리에 정의
  - Props 인터페이스는 컴포넌트 이름 뒤에 `Props` 접미사 사용 (예: `ButtonProps`)
- 컴포넌트는 재사용 가능하도록 설계해야 합니다.
- 상태 관리가 필요한 경우 React Context 또는 Zustand를 사용합니다.
- Next.js의 서버 컴포넌트를 기본으로 사용하고, 클라이언트 컴포넌트가 필요한 경우에만 'use client' 지시어를 사용합니다.
- 이미지는 Next.js의 Image 컴포넌트를 사용해야 합니다.
- API 통신은 `/app/api` 디렉토리에 라우트 핸들러를 정의하여 처리합니다.

## Import Rules

### Absolute Imports

- 모든 임포트는 절대 경로를 사용해야 합니다.
- `tsconfig.json`에 정의된 `@/` 경로 별칭을 사용합니다.

### Import Path Conventions

- Components: `@/components/[category]/[ComponentName]`
  ```typescript
  import { Button } from "@/components/ui/button"
  import { Header } from "@/components/layouts/header"
  import { Card } from "@/components/shared/card"
    • Hooks: @/hooks/[hook-name] 
      typescript
      import { useAuth } from "@/hooks/use-auth"
import { useTheme } from "@/hooks/use-theme"
    • Utils: @/lib/[utility-name] 
      typescript
      import { formatDate } from "@/lib/utils"
import { db } from "@/lib/db"
    • Types: @/types/[type-category] 
      typescript
      import type { User } from "@/types/user"
import type { ApiResponse } from "@/types/api"
    • Config: @/config/[config-name] 
      typescript
      import { siteConfig } from "@/config/site"
import { navLinks } from "@/config/navigation"

    Best Practices

    • 컴포넌트 임포트 시 중복되는 경로 사용을 피하기 위해 index.ts 파일을 활용합니다: 
      typescript
      // Good
import { Button, Card, Input } from "@/components/ui"

// Avoid
import { Button } from "@/components/ui/button"
import { Card } from "@/components/ui/card"
import { Input } from "@/components/ui/input"
    • 타입 임포트는 type 키워드를 사용합니다: 
      typescript
      import type { UserProps } from "@/types/user"
    • 상대 경로 임포트는 사용하지 않습니다: 
      typescript
      // Bad
import { Button } from "../../../components/ui/button"

// Good
import { Button } from "@/components/ui/button"

    Barrel Exports

    • 각 디렉토리는 index.ts 파일을 통해 배럴 익스포트를 제공해야 합니다: 
    typescript
    // /components/ui/index.ts
export * from "./button"
export * from "./card"
export * from "./input"

// Usage
import { Button, Card, Input } from "@/components/ui"

    Route Import Guidelines

    • 페이지 컴포넌트는 해당 라우트 디렉토리 내에서 직접 정의: 
      typescript
      // app/(dashboard)/users/page.tsx
import { DataTable } from "@/components/ui/data-table"
import { getUsers } from "@/lib/data"
    • API 라우트는 @/app/api 내에서 정의: 
      typescript
      // app/api/users/route.ts
import { db } from "@/lib/db"
import { auth } from "@/lib/auth"

1. 일관된 코드 구조를 유지
2. 임포트 경로의 가독성 향상
3. 리팩토링 시 경로 변경의 용이성
4. TypeScript의 경로 별칭 기능 활용
5. 배럴 익스포트를 통한 임포트 구문 간소화

    npx shadcn@latest init -d 에서 설정에 대한 물음없이 자동으로 설치하려면 '--yes'를 붙이면 되지 않나요? '--yes'를 추가해도 설정에 대한 물음이 나오네요

    npx shadcn@latest init의 경우 -y 또는 --yes 플래그가 작동하지 않습니다. 이는 shadcn CLI가 이 플래그를 지원하지 않도록 설계되어 있기 때문입니다.

    대신, shadcn의 초기 설정을 자동화하려면 다음과 같이 모든 옵션을 직접 명령줄 인수로 전달해야 합니다: 

    bash
    npx shadcn@latest init \
  --preset default \
  --style default \
  --typescript \
  --tailwind \
  --components-dir components/ui \
  --utils-dir lib/utils \
  --config shadcn-ui \
  -y

    이렇게 모든 옵션을 미리 지정하면 추가 질문 없이 설치가 진행됩니다.

    기본값들:

    • --preset default: 기본 프리셋 사용
    • --style default: 기본 스타일 사용
    • --typescript: TypeScript 사용
    • --tailwind: Tailwind CSS 사용
    • --components-dir components/ui: UI 컴포넌트 디렉토리 위치
    • --utils-dir lib/utils: 유틸리티 파일 위치
    • --config shadcn-ui: 설정 파일 이름
